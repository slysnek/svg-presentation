SVG

Scalable Vector Graphics

2 types of Images

  • Bitmap(raster)
  • Vector

Different Formats

  • PNG
  • JPEG
  • GIF
  • WebP

WebP

  • Announced by Google in 2010
  • Was intended to replace JPEG, GIF and PNG
  • Supports compression, animation, transparence

Lack of Support

  • Earlier the support of SVG images were limited.
  • On 31 August 2010 Google started to index SVG on the web
  • On 8th December 2010 Google Image Search also began indexing SVG files

Main Difference

  • Bitmap images simply define what color each pixel of the image should be painted.
  • Vector Images provide a set of instructions about how the image should be rendered

When use?

  • Bitmap images are well suited for highly detailed images, such as photographs
  • Vector images are well suited for less detailed images that need to be scaled at different sizes, like icons or data representations.

The problem

  • Bitmap image can be too heavy for low resolution devices
  • Bitmap images tend to be blurry on large resolution devices.

Conclusion

  • SVG images can be created and edited with any text editor
  • SVG images can be searched, indexed, scripted, and compressed
  • SVG images are scalable
  • SVG images can be printed with high quality at any resolution
  • SVG images are zoomable
  • SVG graphics do NOT lose any quality if they are zoomed or resized
  • SVG is an open standard
  • SVG files are pure XML
